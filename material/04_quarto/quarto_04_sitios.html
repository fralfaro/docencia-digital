Sitios web y blogs docentes

Publicar el curso completo como sitio web con Quarto

Un sitio web del curso no es un lujo — es una decisión de diseño pedagógico. Cuando los estudiantes pueden navegar el material en cualquier dispositivo, buscar un concepto dentro del apunte, ver el código y sus resultados en el mismo lugar, y acceder a todo sin descargar nada, el acceso al conocimiento se democratiza de forma concreta.

Quarto produce sitios web profesionales directamente desde archivos .qmd. No necesitas saber HTML, CSS ni JavaScript. El resultado es un sitio navegable, con búsqueda integrada, modo oscuro/claro, y que se despliega en GitHub Pages en minutos.

Este capítulo cubre cómo crear y configurar un sitio de curso completo.

quarto create project website: estructura generada

Crear el proyecto desde la terminal

# Crear un proyecto website nuevo
quarto create project website mat281-sitio

# Alternativamente, con estructura de libro
quarto create project book mat281-apuntes

Quarto genera esta estructura mínima:

mat281-sitio/
├── _quarto.yml          ← configuración central
├── index.qmd            ← página de inicio
├── about.qmd            ← página "Sobre el curso"
└── styles.css           ← estilos CSS base (vacío por defecto)

_quarto.yml: navegación, tema y opciones globales

El archivo _quarto.yml es el cerebro del sitio. Define todo lo que aplica a todas las páginas: la barra de navegación, el sidebar, el tema visual, las opciones de ejecución, y los metadatos globales.

_quarto.yml completo para un sitio de curso

# ══════════════════════════════════════════════════════════════
# _quarto.yml — Sitio de curso MAT281
# ══════════════════════════════════════════════════════════════

project:
  type: website
  output-dir: _site             # carpeta donde se genera el sitio

# ── Metadatos del sitio ────────────────────────────────────────
website:
  title: "MAT281"
  description: "Aplicaciones de la Matemática en Ingeniería — UTFSM 2025-1"
  site-url: "https://fralfaro.github.io/mat281"
  repo-url: "https://github.com/fralfaro/mat281"
  repo-actions: [edit, issue, source]  # botones por página
  favicon: "assets/img/brand/brand__usm.png"
  open-graph: true              # metadata para redes sociales
  twitter-card: true            # card para Twitter/X

  # ── Anuncios ─────────────────────────────────────────────────
  announcement:
    icon: info-circle
    dismissable: true
    content: "**Semana 5:** Material de regresión lineal disponible. Tarea 2 se entrega el viernes 18/04."
    type: info                  # info | warning | danger | success

  # ── Barra de navegación superior ─────────────────────────────
  navbar:
    logo: "assets/img/brand/brand__usm.png"
    logo-alt: "Logo UTFSM"
    title: "MAT281"
    background: primary         # light | dark | primary | secondary | custom

    left:
      - text: "Inicio"
        href: index.qmd

      - text: "Programa"
        href: programa/index.qmd

      - text: "Clases"
        href: lectures/index.qmd

      - text: "Laboratorios"
        href: labs/index.qmd

      - text: "Tareas"
        href: assignments/index.qmd

      - text: "Recursos"
        href: recursos/index.qmd

    right:
      - icon: github
        href: "https://github.com/fralfaro/mat281"
        aria-label: GitHub

      - icon: search
        aria-label: Buscar

      - text: "Colab"
        icon: journal-code
        href: "https://colab.research.google.com/github/fralfaro/mat281/blob/main/"

  # ── Sidebar (menú lateral) ────────────────────────────────────
  sidebar:
    - id: lectures
      title: "Clases"
      style: floating           # floating | docked
      search: true
      collapse-level: 2         # nivel de colapso por defecto
      contents:
        - text: "Todas las clases"
          href: lectures/index.qmd
        - section: "Módulo 1: Estadística Descriptiva"
          contents:
            - lectures/semana01/index.qmd
            - lectures/semana02/index.qmd
        - section: "Módulo 2: Regresión"
          contents:
            - lectures/semana03/index.qmd
            - lectures/semana04/index.qmd
            - lectures/semana05/index.qmd
        - section: "Módulo 3: Clasificación"
          contents:
            - lectures/semana06/index.qmd
            - lectures/semana07/index.qmd

    - id: labs
      title: "Laboratorios"
      style: floating
      contents:
        - text: "Todos los labs"
          href: labs/index.qmd
        - labs/lab01/enunciado.qmd
        - labs/lab02/enunciado.qmd
        - labs/lab03/enunciado.qmd

  # ── Pie de página ─────────────────────────────────────────────
  page-footer:
    left: |
      © 2025 Juan Pérez · Universidad Técnica Federico Santa María
    center: |
      Licencia [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/)
    right: |
      Hecho con [Quarto](https://quarto.org)

  # ── Herramientas por página ───────────────────────────────────
  page-navigation: true         # botones "Anterior / Siguiente"
  back-to-top-navigation: true  # botón volver arriba

# ── Formato HTML global ────────────────────────────────────────
format:
  html:
    # Tema visual
    theme:
      light: [cosmo, assets/scss/light.scss]
      dark:  [darkly, assets/scss/dark.scss]

    # Tabla de contenidos
    toc: true
    toc-depth: 3
    toc-title: "En esta página"

    # Código
    code-fold: false
    code-tools: true
    code-copy: true
    highlight-style: github

    # Figuras
    fig-align: center
    fig-width: 9
    fig-height: 5

    # Tipografía y layout
    number-sections: false
    smooth-scroll: true
    link-external-icon: true
    link-external-newwindow: true
    citations-hover: true
    footnotes-hover: true

# ── Ejecución global ───────────────────────────────────────────
execute:
  freeze: auto
  echo: true
  warning: false
  message: false

# ── Idioma ────────────────────────────────────────────────────
lang: es

# ── Bibliografía global ────────────────────────────────────────
bibliography: assets/bib/referencias.bib
csl: assets/bib/apa-es.csl

# ── Cross-references en español ───────────────────────────────
crossref:
  fig-title:  "Figura"
  tbl-title:  "Tabla"
  eq-title:   "Ecuación"
  fig-prefix: "fig."
  tbl-prefix: "tbl."
  eq-prefix:  "ec."
  sec-prefix: "sec."

Configuración del sidebar por sección

Quarto permite asignar sidebars diferentes a diferentes secciones del sitio:

website:
  sidebar:
    - id: lectures-sidebar
      title: "Clases"
      contents:
        - lectures/index.qmd
        - section: "Módulo 1"
          contents:
            - lectures/semana01/index.qmd

    - id: labs-sidebar
      title: "Labs"
      contents:
        - labs/index.qmd
        - labs/lab01/enunciado.qmd

En cada página, especificas qué sidebar usar:

# En lectures/semana01/index.qmd
---
sidebar: lectures-sidebar
---

Listing pages: blog, portafolio, catálogo de recursos

Las páginas de listado muestran automáticamente una colección de documentos como tarjetas, tabla o grilla. Son perfectas para:

  • Catálogo de clases (una tarjeta por semana)
  • Listado de laboratorios con su estado (entregado/pendiente)
  • Blog del curso con anuncios y novedades
  • Portafolio de proyectos estudiantiles

Listing básico de clases

# lectures/index.qmd
---
title: "Clases"
listing:
  type: grid             # default | table | grid
  contents: "semana*"    # glob para incluir carpetas que empiecen con "semana"
  sort: "date asc"       # ordenar por fecha ascendente
  fields: [date, title, description, categories]
  image-height: 200px
  grid-columns: 3
  filter-ui: true        # selector de filtro
  sort-ui: true          # selector de ordenamiento
---

Cada documento en semana*/index.qmd necesita estos campos en su frontmatter:

# lectures/semana03/index.qmd
---
title: "Semana 3: Regresión Lineal Simple"
date: "2025-04-08"
description: "Estimación MCO, $R^2$, diagnóstico de residuos."
categories: [regresión, estadística, python]
image: "preview.png"
---

Tipos de listing

type: default — Lista vertical con imagen a la izquierda:

listing:
  type: default
  contents: "semana*"
  fields: [date, title, description, author]
  sort: "date desc"

type: grid — Tarjetas en cuadrícula:

listing:
  type: grid
  contents: "semana*"
  grid-columns: 3
  image-height: 180px
  fields: [date, title, description, categories, image]

type: table — Tabla con columnas configurables:

listing:
  type: table
  contents: "semana*"
  fields: [date, title, categories, reading-time]
  date-format: "DD/MM/YYYY"
  field-display-names:
    date: "Fecha"
    title: "Tema"
    categories: "Módulo"
    reading-time: "Tiempo estimado"

Listing como blog del curso

Para anuncios y novedades del curso:

# noticias/index.qmd
---
title: "Novedades del curso"
listing:
  type: default
  contents: "posts"
  sort: "date desc"
  fields: [date, title, description, categories]
  feed: true             # genera feed RSS
  categories: true       # muestra filtro por categorías
  date-format: "DD MMMM YYYY"

Barra de búsqueda integrada

Quarto incluye búsqueda de texto completo sin ningún servidor externo — funciona como JavaScript puro en el navegador usando un índice generado al momento del build.

Configurar la búsqueda

website:
  search:
    location: navbar        # navbar | sidebar
    type: overlay           # overlay | textbox
    limit: 20               # máximo de resultados
    collapse-after: 3       # colapsar resultados por sección
    copy-button: true       # botón para copiar el resultado

    # Fuentes a indexar
    algolia:                # opcional: Algolia DocSearch (más potente)
      index-name: mat281
      application-id: TU_APP_ID
      search-only-api-key: TU_KEY

La búsqueda por defecto (sin Algolia) indexa automáticamente todo el contenido de texto del sitio, incluyendo:

  • Títulos y subtítulos
  • Texto de los párrafos
  • Captions de figuras y tablas
  • Contenido de callouts
  • Código fuente (si está visible)

Excluir páginas del índice de búsqueda

# En el frontmatter de una página específica
---
search: false    # no indexar esta página
---
# En _quarto.yml — excluir patrones de rutas
website:
  search:
    exclude:
      - "404.qmd"
      - "assets/**"
      - "examenes/**"       # no indexar exámenes

Búsqueda multiidioma

Si el sitio tiene páginas en español e inglés:
