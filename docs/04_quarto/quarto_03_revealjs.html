Presentaciones con RevealJS

Slides académicas con código, ecuaciones y notas del presentador

RevealJS es el sistema de presentaciones HTML de Quarto. A diferencia de PowerPoint (donde el archivo .pptx es el resultado) o Beamer (donde el .pdf es el resultado), una presentación RevealJS es un sitio web — se ejecuta en el navegador, puede contener código vivo, animaciones, audio, video, y se puede publicar directamente en GitHub Pages junto con el resto del material del curso.

Para un docente de ingeniería o matemáticas, la ventaja más concreta es esta: el mismo archivo .qmd que usas para el apunte de la semana también puede producir las slides de esa semana. Mantienes un solo lugar de verdad.

Crear slides desde un archivo .qmd

El frontmatter mínimo

Una presentación RevealJS es un archivo .qmd con format: revealjs en el frontmatter:

---
title: "Regresión Lineal Simple"
subtitle: "MAT281 · Semana 5"
author: "Prof. Juan Pérez"
date: "Abril 2025"
format: revealjs
---

Con solo eso, Quarto renderiza el archivo como una presentación HTML completa.

# Renderizar la presentación
quarto render semana05_slides.qmd

# Preview en vivo (se actualiza al guardar)
quarto preview semana05_slides.qmd

Estructura completa del frontmatter

---
title: "Regresión Lineal Simple"
subtitle: "MAT281 · Semana 5"
author: "Prof. Juan Pérez"
institute: "Departamento de Matemáticas · UTFSM"
date: "Abril 2025"
lang: es

format:
  revealjs:
    # ── Tema visual ────────────────────────────────────────
    theme: [default, assets/scss/slides.scss]
    # Temas: default, dark, moon, sky, blood, night,
    #         league, beige, serif, simple, solarized

    # ── Identidad ─────────────────────────────────────────
    logo: assets/img/brand/brand__usm.png
    footer: "MAT281 — 2025-1 · [Volver al sitio](../../index.html)"

    # ── Dimensiones ────────────────────────────────────────
    width: 1280
    height: 720
    margin: 0.1          # margen exterior como fracción del tamaño

    # ── Numeración ─────────────────────────────────────────
    slide-number: "c/t"  # current/total
    # Opciones: true | false | "c" | "t" | "c/t" | "h/v" | "h.v"

    # ── Navegación ─────────────────────────────────────────
    controls: true        # flechas de navegación
    progress: true        # barra de progreso inferior
    history: true         # navegación con botón Atrás del browser
    keyboard: true        # atajos de teclado
    overview: true        # vista general con Esc

    # ── Transiciones ──────────────────────────────────────
    transition: slide     # none | fade | slide | convex | concave | zoom
    transition-speed: default  # default | fast | slow
    background-transition: fade  # transición del fondo entre slides

    # ── Código ────────────────────────────────────────────
    highlight-style: github
    code-line-numbers: true    # número de línea en bloques de código
    code-copy: true

    # ── Herramientas de presentador ────────────────────────
    chalkboard: true           # pizarrón virtual (tecla B)
    pointer: true              # puntero láser virtual (tecla Q)

    scrollable: false    # true = scroll en slides que no caben

    # ── Auto-animación (morph) ─────────────────────────────
    auto-animate: true   # animación automática entre slides con mismo id

execute:
  echo: true
  warning: false
  message: false
  freeze: auto

Separadores de slide: ## y ---

Cómo funciona la división en slides

Quarto convierte el Markdown en slides usando dos tipos de separadores:

Regla general con slide-level: 2 (el valor por defecto):

  • # → Slide de sección (fondo completo, sin contenido adicional)
  • ## → Slide normal (con título)
  • --- → Slide sin título (útil para imágenes o contenido de pantalla completa)

Fragmentos, animaciones e incremental bullets

Incremental bullets

Para revelar bullets uno a uno:

# Globalmente para toda la presentación
format:
  revealjs:
    incremental: true
# Solo para una lista específica (aunque el global esté en false)
::: {.incremental}
- Primer punto que aparece solo
- Segundo punto que aparece al avanzar
- Tercer punto al avanzar de nuevo
:::
# Desactivar incremental para una lista específica (cuando el global es true)
::: {.nonincremental}
- Este aparece de golpe
- Con todos los puntos juntos
:::

Fragments: revelar cualquier contenido paso a paso

Los fragmentos funcionan para cualquier elemento — no solo listas:

Este texto está visible desde el principio.

::: {.fragment}
Este párrafo aparece al avanzar.
:::

::: {.fragment .fade-in}
Este aparece con fade-in.
:::

::: {.fragment .highlight-red}
Este texto existe desde el principio, pero se resalta al avanzar.
:::

::: {.fragment .fade-out}
Este existe desde el principio y desaparece al avanzar.
:::

Clases de animación disponibles para fragments:

Clase Efecto
.fade-in Aparece con desvanecimiento (por defecto)
.fade-up Aparece subiendo desde abajo
.fade-down Aparece bajando desde arriba
.fade-left Aparece desde la derecha
.fade-right Aparece desde la izquierda
.fade-in-then-out Aparece y luego desaparece
.fade-in-then-semi-out Aparece y luego se vuelve transparente
.highlight-red Se resalta en rojo
.highlight-blue Se resalta en azul
.highlight-green Se resalta en verde
.strike Aparece tachado
.grow Crece de tamaño
.shrink Encoge

Código ejecutable en la presentación (WASM)

RevealJS puede ejecutar código Python directamente en el navegador usando WebAssembly (Pyodide). No requiere servidor:

```{pyodide-python}
# Este código se ejecuta EN el navegador durante la presentación
import numpy as np

x = np.array([1, 2, 3, 4, 5])
y = 2 * x + np.random.normal(0, 0.5, 5)
print(f"Correlación: {np.corrcoef(x, y)[0,1]:.4f}")
```

Para activarlo, agrega en el frontmatter:

format:
  revealjs:
    # No se necesita configuración extra — Quarto lo detecta automáticamente
    # Solo usa ```{pyodide-python} en lugar de ```python
Limitaciones del modo WASM

El modo pyodide-python carga Pyodide (~20MB) en el navegador. Solo funciona con librerías puras Python — pandas, numpy, scikit-learn funcionan, pero no librerías con código nativo como lightgbm o algunos backends de matplotlib. Úsalo para demos simples e interactivos, no para análisis complejos.

Exportar a PDF para entregar

Método 1: desde el navegador (más simple)

  1. Abre la presentación en Chrome o Chromium
  2. Agrega ?print-pdf al final de la URL: presentacion.html?print-pdf
  3. La presentación se reformatea para impresión
  4. Ctrl+P → Guardar como PDF
  5. Configurar: Sin márgenes, Tamaño A4 o Letter, Fondo gráfico activado

Método 2: con Quarto y Chrome headless (automatizable)

# Quarto usa Chrome headless para el PDF
quarto render semana05_slides.qmd --to revealjs

# Convertir el HTML a PDF automáticamente
# (requiere Chrome instalado)
quarto render semana05_slides.qmd --to pdf

Esto funciona porque Quarto puede usar Chrome en modo headless para capturar el HTML como PDF.

Método 3: definir ambos formatos en el frontmatter

format:
  revealjs:
    theme: [default, assets/scss/slides.scss]
    slide-number: true
    # ... resto de opciones ...

  pdf:
    documentclass: beamer       # usa la clase Beamer de LaTeX
    aspectratio: 169            # 16:9
    theme: default
quarto render semana05_slides.qmd --to pdf

Método 4: con decktape (máximo control)

# Instalar decktape
npm install -g decktape

# Convertir a PDF (con todas las animaciones capturadas)
decktape reveal semana05_slides.html semana05_slides.pdf

# Con tamaño personalizado
decktape reveal --size 1280x720 semana05_slides.html semana05_slides.pdf

# Solo capturar ciertos slides
decktape reveal --slides 1-10 semana05_slides.html primeros10.pdf

Ajustar para impresión en PDF

Algunos elementos de HTML no se ven bien en PDF. Usa CSS condicional:

// assets/scss/slides.scss
// Solo en modo print-pdf
@media print {
  .reveal .slide-footer {
    display: none;    // ocultar footer en PDF
  }

  .reveal pre code {
    font-size: 0.65em;    // código más pequeño en papel
  }

  // Asegurar que el fondo se imprime
  * {
    -webkit-print-color-adjust: exact;
    print-color-adjust: exact;
  }
}

Parte II: El modelo

Definición formal

Modelo de regresión lineal simple:

\[Y_i = \beta_0 + \beta_1 X_i + \varepsilon_i, \qquad i = 1, \ldots, n\]

donde \(\varepsilon_i \sim \mathcal{N}(0, \sigma^2)\) i.i.d.

¿Qué significa cada término?

  • \(Y_i\): variable respuesta (nota final del estudiante \(i\))
  • \(X_i\): variable predictora (horas de estudio del estudiante \(i\))
  • \(\beta_0, \beta_1\): parámetros desconocidos — los estimamos con los datos
  • \(\varepsilon_i\): error aleatorio — todo lo que el modelo no captura

Enfatizar que \(\beta_0\) y \(\beta_1\) son FIJOS pero desconocidos. Los \(\varepsilon_i\) son aleatorios — por eso la nota no es perfectamente predecible.

Los supuestos del modelo

Estos supuestos se deben verificar siempre — no asumirlos.

  1. Linealidad: \(\mathbb{E}[Y_i | X_i] = \beta_0 + \beta_1 X_i\)
  2. Exogeneidad: \(\mathbb{E}[\varepsilon_i | X_i] = 0\)
  3. Homocedasticidad: \(\text{Var}(\varepsilon_i | X_i) = \sigma^2\) constante
  4. Independencia: \(\varepsilon_i\) y \(\varepsilon_j\) son independientes (\(i \neq j\))
  5. Normalidad (opcional): \(\varepsilon_i \sim \mathcal{N}(0, \sigma^2)\)

Parte III: Estimación

Mínimos cuadrados ordinarios (MCO)

Objetivo: encontrar \(\hat{\beta}_0\) y \(\hat{\beta}_1\) que minimicen la SCR:

\[\text{SCR}(\beta_0, \beta_1) = \sum_{i=1}^{n}(Y_i - \beta_0 - \beta_1 X_i)^2\]

Solución: tomar derivadas e igualar a cero.

\[\hat{\beta}_1 = \frac{S_{XY}}{S_{XX}} = \frac{\sum(X_i - \bar{X})(Y_i - \bar{Y})}{\sum(X_i - \bar{X})^2}\]

\[\hat{\beta}_0 = \bar{Y} - \hat{\beta}_1 \bar{X}\]

Mostrar la derivación en la pizarra si hay tiempo (unos 8 minutos). Si no hay tiempo, decir que está en el apunte de la semana 5. La clave es que es un sistema de 2 ecuaciones con 2 incógnitas.

Demo: estimación en Python

#| code-line-numbers: "|1-2|4-7|9-12"
import numpy as np

# Datos
X = np.array([4, 6, 8, 5, 10, 7, 3, 9])
y = np.array([4.5, 5.2, 5.8, 4.9, 6.5, 5.5, 3.8, 6.1])
X_bar, y_bar = X.mean(), y.mean()

# Estimadores MCO
S_XY  = np.sum((X - X_bar) * (y - y_bar))
S_XX  = np.sum((X - X_bar)**2)
beta_1 = S_XY / S_XX
beta_0 = y_bar - beta_1 * X_bar

Demo: resultado

#| code-line-numbers: "1-12|14-15"
import numpy as np

X = np.array([4, 6, 8, 5, 10, 7, 3, 9])
y = np.array([4.5, 5.2, 5.8, 4.9, 6.5, 5.5, 3.8, 6.1])
X_bar, y_bar = X.mean(), y.mean()

S_XY  = np.sum((X - X_bar) * (y - y_bar))
S_XX  = np.sum((X - X_bar)**2)
beta_1 = S_XY / S_XX
beta_0 = y_bar - beta_1 * X_bar

# Resultado
print(f"β̂₀ = {beta_0:.4f}")
print(f"β̂₁ = {beta_1:.4f}")
print(f"Modelo: Ŷ = {beta_0:.2f} + {beta_1:.2f}·X")

Parte IV: Bondad de ajuste

El coeficiente \(R^2\)

\[R^2 = 1 - \frac{\text{SCR}}{\text{SCT}} = 1 - \frac{\sum(Y_i - \hat{Y}_i)^2}{\sum(Y_i - \bar{Y})^2}\]

  • \(R^2 \in [0, 1]\) — siempre (para MCO con intercepto)
  • \(R^2 = 0\): el modelo no explica nada más que la media
  • \(R^2 = 1\): ajuste perfecto, todos los puntos sobre la recta
  • \(R^2 = 0.71\): el modelo explica el 71% de la variabilidad de \(Y\)

\(R^2\) alto no garantiza un buen modelo. Siempre verificar los residuos.

Parte V: Diagnóstico

Los cuatro gráficos de diagnóstico

#| echo: false
#| fig-width: 5
#| fig-height: 4.5
from sklearn.linear_model import LinearRegression
from scipy import stats
import matplotlib.pyplot as plt

modelo = LinearRegression()
modelo.fit(X.reshape(-1,1), y)
y_pred = modelo.predict(X.reshape(-1,1))
residuos = y - y_pred

fig, axes = plt.subplots(2, 2, figsize=(5, 4.5))

# (1) Residuos vs ajustados
axes[0,0].scatter(y_pred, residuos, s=40, alpha=0.8, color='#FF9800')
axes[0,0].axhline(0, color='red', ls='--')
axes[0,0].set_title('Residuos vs Ajustados', fontsize=9)
axes[0,0].set_xlabel('Å¶', fontsize=8)

# (2) Q-Q plot
stats.probplot(residuos, plot=axes[0,1])
axes[0,1].set_title('Q-Q Normal', fontsize=9)

# (3) Scale-location
axes[1,0].scatter(y_pred, np.sqrt(np.abs(residuos)), s=40, alpha=0.8, color='#4CAF50')
axes[1,0].set_title('Scale-Location', fontsize=9)

# (4) Residuos vs leverage (simplificado)
axes[1,1].scatter(range(len(residuos)), residuos, s=40, alpha=0.8, color='#9C27B0')
axes[1,1].axhline(0, color='red', ls='--')
axes[1,1].set_title('Residuos vs Orden', fontsize=9)

for ax in axes.flat:
    ax.spines[['top','right']].set_visible(False)
    ax.grid(alpha=0.3)
    ax.tick_params(labelsize=7)

plt.tight_layout()
plt.show()

¿Qué buscamos?

  • Residuos vs Ajustados: nube sin patrón → homocedasticidad ✅
  • Q-Q Plot: puntos sobre la línea → normalidad ✅
  • Scale-Location: banda horizontal → varianza constante ✅
  • vs Orden: sin tendencia → independencia ✅
