Plantillas y producción de documentos

Beamer, tikz, IA para LaTeX y la decisión LaTeX vs Quarto

Los dos capítulos anteriores cubrieron LaTeX desde los fundamentos hasta la escritura científica. Este capítulo cubre la capa de producción: cómo usar plantillas institucionales y de revistas para no partir de cero, cómo hacer presentaciones académicas con Beamer, qué paquetes resuelven los casos más frecuentes, cómo usar IA para generar código LaTeX sin memorizarlo todo, y — la pregunta más práctica — cuándo usar LaTeX y cuándo usar Quarto.

Plantillas institucionales (UTFSM, IEEE, APA)

Por qué usar plantillas

Una plantilla LaTeX es un archivo .tex preconfigurado con el formato correcto: márgenes, tipografía, estilos de sección, encabezados, pie de página, portada institucional. En lugar de configurar todo desde cero (y cometer errores de formato), partes del resultado final y solo escribes el contenido.

Para docentes, las plantillas más relevantes son:

Tipo Para qué Fuente
Institucional UTFSM Informes, memorias, guías de curso DTD / Biblioteca UTFSM
IEEE Papers para conferencias y revistas de ingeniería overleaf.com/gallery/tagged/ieee
APA 7ª ed. Trabajos en ciencias sociales y educación Paquete apa7
ACM Computación y sistemas overleaf.com/gallery/tagged/acm
LNCS Springer Lecture Notes in Computer Science overleaf.com/gallery/tagged/springer

Beamer: presentaciones académicas en LaTeX

Por qué Beamer para docencia universitaria

Beamer produce presentaciones PDF con tipografía y ecuaciones de la misma calidad que el resto de tus documentos LaTeX. La ventaja principal para docentes de matemáticas e ingeniería: las ecuaciones se renderizan perfectamente, con el mismo estilo que en los apuntes y las guías.

\section{El modelo}

\begin{frame}{Modelo de regresión lineal simple}

  \begin{block}{Definición}
    La regresión lineal simple modela:
    \begin{equation}
      Y_i = \beta_0 + \beta_1 X_i + \varepsilon_i
    \end{equation}
    donde $\varepsilon_i \sim \mathcal{N}(0, \sigma^2)$ i.i.d.
  \end{block}

  \begin{alertblock}{Supuestos críticos}
    \begin{itemize}
      \item Linealidad: $\mathbb{E}[Y|X] = \beta_0 + \beta_1 X$
      \item Homocedasticidad: $\text{Var}(\varepsilon_i) = \sigma^2$ constante
      \item Independencia: $\varepsilon_i$ independientes entre sí
    \end{itemize}
  \end{alertblock}

  \begin{exampleblock}{En la práctica}
    Estos supuestos se verifican con gráficos de residuos.
  \end{exampleblock}

\end{frame}

algorithm2e: pseudocódigo y algoritmos

\usepackage[linesnumbered, ruled, vlined, spanish]{algorithm2e}

\begin{algorithm}[H]
  \caption{Algoritmo de Descenso de Gradiente}
  \label{alg:gradient_descent}

  \KwIn{$f(\theta)$: función de costo, $\alpha$: tasa de aprendizaje,
        $\varepsilon$: tolerancia, $\theta_0$: parámetros iniciales}
  \KwOut{$\theta^*$: parámetros óptimos}

  \BlankLine
  $\theta \leftarrow \theta_0$\;
  $t \leftarrow 0$\;

  \While{$\|\nabla f(\theta)\| > \varepsilon$}{
    $g \leftarrow \nabla_\theta f(\theta)$
      \tcp*[r]{calcular gradiente}
    $\theta \leftarrow \theta - \alpha \cdot g$
      \tcp*[r]{actualizar parámetros}
    $t \leftarrow t + 1$\;

    \If{$t > T_{\max}$}{
      \textbf{break} \tcp*[r]{máximo de iteraciones}
    }
  }

  \Return{$\theta$}
\end{algorithm}

Uso de IA para generar código LaTeX

Por qué la IA es especialmente útil para LaTeX

LaTeX tiene una sintaxis extensa y a veces críptica. Nadie memoriza todos los parámetros de pgfplots o las opciones de tcolorbox. La IA (Claude, ChatGPT) puede generar código LaTeX correcto en segundos — lo que antes requería consultar la documentación durante minutos.

La combinación más efectiva: tú describes lo que necesitas, la IA genera el código, tú lo pegas en Overleaf y verificas que compila.

Limitaciones de la IA con LaTeX

Lo que la IA hace mal en LaTeX
  • Paquetes incompatibles: puede sugerir combinaciones de paquetes que generan conflictos. Si algo no compila, pide a la IA que identifique el conflicto.
  • Rutas de archivos: no sabe dónde están tus imágenes. Siempre debes ajustar los paths.
  • Versiones de paquetes: puede usar sintaxis de una versión diferente a la de Overleaf. Especifica \pgfplotsset{compat=1.18} si usas pgfplots.
  • Tikz complejo: el código tikz generado por IA suele requerir ajustes de coordenadas y espaciados. Úsalo como punto de partida, no como resultado final.
  • Caracteres especiales en español: puede olvidar \usepackage[spanish]{babel} o usar comillas incorrectas.

Sincronizar Overleaf con GitHub

Por qué sincronizar

La sincronización Overleaf ↔︎ GitHub combina lo mejor de ambos mundos:

  • Overleaf: edición colaborativa visual, compilación instantánea, sin instalación.
  • GitHub: control de versiones completo, CI/CD, integración con el sitio del curso.

Configurar la sincronización

Requisitos: cuenta Overleaf Pro (o Education). La versión gratuita no tiene sincronización con GitHub.

Paso 1 — Desde un proyecto nuevo en Overleaf:

  1. Menu (☰) → GitHub
  2. Create a GitHub repository
  3. Selecciona la organización y nombre del repositorio
  4. Overleaf crea el repositorio y hace el primer push

Paso 2 — Desde un repositorio GitHub existente:

  1. Menu (☰) → GitHub
  2. Link an existing GitHub repository
  3. Busca y selecciona el repositorio

Paso 3 — Flujo diario:

Editas en Overleaf
    ↓
Menu → GitHub → "Push Overleaf changes to GitHub"
    ↓
El repositorio GitHub se actualiza con los cambios de Overleaf

---

Alguien hace cambios en GitHub (push directo, merge de PR)
    ↓
Menu → GitHub → "Pull GitHub changes into Overleaf"
    ↓
Overleaf recibe los cambios del repositorio

GitHub Action para compilar LaTeX

# .github/workflows/compile-latex.yml
name: Compilar LaTeX

on:
  push:
    branches: [main]
    paths:
      - '**.tex'
      - '**.bib'

jobs:
  compilar:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Compilar todos los PDFs
        uses: xu-cheng/latex-action@v3
        with:
          root_file: |
            guias/guia01/main.tex
            guias/guia02/main.tex
            apuntes/main.tex
          latexmk_use_lualatex: true   # o false para pdflatex
          work_in_root_file_dir: true

      - name: Subir PDFs como artefactos
        uses: actions/upload-artifact@v4
        with:
          name: pdfs-compilados
          path: |
            guias/guia01/main.pdf
            guias/guia02/main.pdf
            apuntes/main.pdf
          retention-days: 30

LaTeX vs Quarto: criterios para elegir según el caso

Esta es la pregunta más práctica de este capítulo. Ambas herramientas producen documentos de alta calidad — pero sirven propósitos distintos y tienen perfiles de usuario diferentes.

La tabla de decisión

Criterio LaTeX (Overleaf) Quarto
Ecuaciones matemáticas Excelente — el estándar Buena (usa LaTeX internamente)
Código ejecutable No (usa listings para mostrar código estático) Sí — el código corre y genera outputs
Publicación web Limitado (requiere herramientas adicionales) Nativo — HTML, RevealJS, sitios completos
Colaboración con no-técnicos Difícil (requieren conocer LaTeX) Más fácil (Markdown es más intuitivo)
Plantillas de revistas Amplísimo catálogo (IEEE, APA, Springer…) Limitado (algunas extensiones)
Control tipográfico Total — cada milímetro Parcial — depende del tema
Figuras TikZ/PGFPlots Sí, nativo No (se incluyen como imágenes)
Curva de aprendizaje Alta Media
Reproducibilidad Manual (código en listings) Automática (chunks ejecutados)
Output a Word Complicado Nativo
Actualizaciones frecuentes Requiere recompilar manualmente quarto preview actualiza en vivo

El árbol de decisión

Casos concretos y su herramienta ideal

→ Guía de ejercicios de matemáticas: LaTeX. Ecuaciones numeradas, referencias cruzadas, formato de examen. El estudiante imprime o lee el PDF.

→ Apuntes de un curso de programación: Quarto. El código corre, los outputs se generan automáticamente, se publica como sitio web.

→ Paper para CLEI o IEEE Access: LaTeX con plantilla IEEEtran. La revista lo requiere explícitamente.

→ Reporte de análisis de datos: Quarto. El análisis es el código; el reporte es reproducible; el output puede ser PDF y HTML.

→ Tesis de pregrado (UTFSM): LaTeX. Las memorias tienen requisitos de formato precisos que LaTeX maneja mejor.

→ Slides de una clase teórica (con muchas ecuaciones): Beamer. Consistencia con el material escrito, ecuaciones perfectas.

→ Slides de una clase práctica (con demos de código): Quarto RevealJS. El código se ejecuta en vivo durante la presentación.

→ Syllabus y material administrativo del curso: Quarto o incluso Markdown puro. No hay ecuaciones, y la publicación en web es prioritaria.

La respuesta pragmática

Para la mayoría de los docentes universitarios de ingeniería y matemáticas en 2025, la respuesta es: usa ambas.

LaTeX para los documentos que deben verse perfectos en papel y que tienen muchas ecuaciones: guías, pruebas, apuntes de matemáticas, papers.

Quarto para el material que vive en la web o que incluye código ejecutado: sitio del curso, notebooks, reportes de datos, slides interactivas.

No son rivales — son complementarias. Y Quarto usa la sintaxis de LaTeX para ecuaciones, así que aprender LaTeX también mejora tu Quarto.

Resumen de la decisión en una línea por criterio

Ecuaciones perfectas y tipografía precisa      → LaTeX
Código que genera outputs                      → Quarto
Revista académica con plantilla                → LaTeX
Sitio web del curso                            → Quarto
Examen o guía para imprimir                   → LaTeX
Notebook reproducible                          → Quarto
Diagrama TikZ complejo                         → LaTeX
Slide con demo de código en vivo               → Quarto RevealJS
Colaboración con persona que no sabe LaTeX     → Quarto (o Google Docs)
