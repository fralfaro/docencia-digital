Las dos limitaciones fundamentales de los LLMs

Los modelos de lenguaje tienen dos limitaciones para uso docente:

1. Fecha de corte:

No saben nada posterior a su entrenamiento. Para un curso de ingeniería con papers recientes, esto importa.

2. Sin acceso a tu material específico:

No conocen tu syllabus, tus apuntes, tus rúbricas, tu forma de explicar los conceptos.

La solución naïve: incluir todo el material en el prompt.

El problema: los prompts tienen límites de tamaño, son costosos, y el modelo tiene dificultad para recuperar información específica de documentos muy largos.

La solución: RAG

Retrieval Augmented Generation — recuperar los fragmentos relevantes de tus documentos justo antes de generar la respuesta.

El modelo solo ve lo que es relevante para tu pregunta, no todo el corpus.

¿Cómo funciona RAG?

La clave está en los embeddings

Un embedding es una representación vectorial del texto. Los fragmentos semánticamente similares a la pregunta quedan cerca en el espacio vectorial — así el sistema recupera los más relevantes.

Opción sin código: NotebookLM

NotebookLM es RAG sin código:

  1. Subes tus documentos
  2. Haces preguntas
  3. Recibes respuestas con citas

Para docencia: es suficiente para la mayoría de los casos de uso.

Sus limitaciones:

  • No puedes personalizarlo
  • No puedes integrarlo en tu sitio web
  • No puedes configurar el comportamiento del asistente
  • No puedes agregar fuentes dinámicas (como una base de datos que cambia)

Cuando estas limitaciones importan → es momento de construir tu propio RAG.

Otras herramientas sin código:

Herramienta Para qué
NotebookLM Análisis documental, guías de estudio
Poe.com Crear chatbots con personalidad personalizada
Botpress Chatbots con flujos de conversación
Microsoft Copilot Studio Agentes institucionales con Office 365

Para docentes no técnicos

NotebookLM + Copilot Studio cubren el 90% de los casos de uso sin escribir ninguna línea de código.

RAG con código: los componentes

Stack mínimo viable:

# 1. Cargar y dividir documentos
from langchain.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

loader = PyPDFLoader("apuntes_mat281.pdf")
docs = loader.load()

splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
chunks = splitter.split_documents(docs)

Por qué dividir en chunks:

Los modelos de embedding tienen límites de tokens. Los chunks permiten recuperar solo los fragmentos relevantes, no el documento entero.

# 2. Crear embeddings y base vectorial
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import Chroma

embeddings = OpenAIEmbeddings()

vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./db_mat281"
)

# 3. Crear el retriever
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 4}  # top 4 fragmentos más relevantes
)

# 4. Crear la cadena RAG
from langchain.chains import RetrievalQA

qa = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=retriever
)

RAG con código: la cadena completa

from langchain_openai import ChatOpenAI
from langchain.prompts import PromptTemplate
from langchain.chains import RetrievalQA

# Modelo de lenguaje
llm = ChatOpenAI(model="gpt-4o", temperature=0)

# Template del prompt
template = """Eres un asistente de estudio para
el curso MAT281 de la UTFSM.

Responde la pregunta del estudiante usando
ÚNICAMENTE la siguiente información del curso.
Si la respuesta no está en el material,
dilo claramente.

Contexto del curso:
{context}

Pregunta del estudiante:
{question}

Responde en español, de forma clara y concisa.
"""

prompt = PromptTemplate(
    input_variables=["context", "question"],
    template=template
)
# Cadena RAG completa
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=retriever,
    chain_type_kwargs={"prompt": prompt},
    return_source_documents=True
)

# Usar el asistente
resultado = qa_chain.invoke(
    "¿Cuál es la diferencia entre Ridge y Lasso?"
)

print("Respuesta:", resultado["result"])
print("\nFuentes:")
for doc in resultado["source_documents"]:
    print(f"  - {doc.metadata['source']} p.{doc.metadata['page']}")

Output:

Respuesta: La regresión Ridge usa penalización L2...
Fuentes:
  - apuntes_mat281.pdf p.47
  - apuntes_mat281.pdf p.52

Alternativa local: Ollama + LlamaIndex

Para quien no quiere usar APIs de pago o necesita privacidad de datos:

# Instalar Ollama
# ollama.ai → descarga para tu OS

# Descargar modelo (corre localmente)
ollama pull llama3.2
ollama pull nomic-embed-text  # para embeddings
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from llama_index.llms.ollama import Ollama
from llama_index.embeddings.ollama import OllamaEmbedding

# Cargar documentos
documentos = SimpleDirectoryReader("./apuntes/").load_data()

# Crear índice con modelo local
llm = Ollama(model="llama3.2", request_timeout=120)
embed = OllamaEmbedding(model_name="nomic-embed-text")

index = VectorStoreIndex.from_documents(documentos,
    llm=llm, embed_model=embed)

# Consultar
query_engine = index.as_query_engine()
response = query_engine.query("¿Qué es el gradiente descendente?")

Ventajas de local:

  • Sin costo por token
  • Sin límites de velocidad
  • Datos nunca salen de tu computador (privacidad total)
  • Funciona sin internet

Desventajas:

  • Requiere GPU potente para velocidad razonable
  • Modelos locales son menos capaces que GPT-4o o Claude
  • Más complejo de configurar

Para documentos sensibles

Si el material del curso contiene información de estudiantes u otros datos sensibles, un RAG completamente local elimina el riesgo de que los datos lleguen a servidores externos.

Cuándo construir tu propio RAG vs usar NotebookLM

Criterio NotebookLM RAG propio
Complejidad técnica Ninguna Media-alta
Personalización Limitada Total
Integración en sitio web No Sí
Privacidad total de datos No (Google) Sí (local)
Fuentes dinámicas Manual Automático
Costo Gratis/Pro Google API + hosting
Tiempo de puesta en marcha Minutos Días/semanas

La recomendación práctica

Empieza con NotebookLM. Si las limitaciones te bloquean → considera LangChain o LlamaIndex. Si necesitas privacidad total → Ollama local.

La mayoría de los docentes universitarios nunca necesitará ir más allá de NotebookLM.
