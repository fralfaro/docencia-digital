Por qué un sitio web del curso importa

Un sitio web del curso no es un lujo — es una decisión de diseño pedagógico.

Cuando los estudiantes pueden:

  • Navegar el material en cualquier dispositivo
  • Buscar un concepto dentro del apunte
  • Ver el código y sus resultados en el mismo lugar
  • Acceder a todo sin descargar nada

… el acceso al conocimiento se democratiza de forma concreta.

Quarto produce sitios web profesionales sin saber HTML, CSS ni JavaScript.

El resultado

  • Búsqueda integrada en todo el sitio
  • Modo oscuro/claro automático
  • Navegación con sidebar
  • Responsive (funciona en móvil)
  • Se despliega en GitHub Pages en minutos

Crear el proyecto website

# Crear un proyecto website nuevo
quarto create project website mat281-sitio

Estructura generada:

mat281-sitio/
├── _quarto.yml       # configuración central del sitio
├── index.qmd         # página de inicio (home)
├── about.qmd         # página "acerca de" (ejemplo)
└── styles.css        # estilos adicionales

El archivo _quarto.yml es el cerebro del sitio

Ahí defines la estructura de navegación, el tema visual, el footer, y las opciones de ejecución de código para todo el sitio. Los archivos .qmd individuales heredan esas opciones.

_quarto.yml para el sitio del curso

project:
  type: website

website:
  title: "MAT281 — 2025-1"
  description: "Aplicaciones de la Matemática en Ingeniería"
  favicon: assets/img/brand/brand__usm.png

  navbar:
    logo: assets/img/brand/brand__usm.png
    left:
      - text: "Inicio"
        href: index.qmd
      - text: "Clases"
        href: lectures/index.qmd
      - text: "Laboratorios"
        href: labs/index.qmd
      - text: "Tareas"
        href: assignments/index.qmd
    right:
      - icon: github
        href: https://github.com/tu-username/mat281-2025-1
      - icon: search
        text: Buscar

  page-footer:
    center: |
      © 2025 Juan Pérez · UTFSM ·
      [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/)

Navegación con sidebar

Sidebar automático por sección:

website:
  sidebar:
    style: floating
    search: true
    contents:
      - section: "Semana 1 — Introducción"
        contents:
          - lectures/semana01/intro.qmd
          - lectures/semana01/notebook.qmd
      - section: "Semana 2 — Estadística"
        contents:
          - lectures/semana02/teoria.qmd
          - lectures/semana02/notebook.qmd
      - section: "Laboratorios"
        contents:
          - labs/lab01.qmd
          - labs/lab02.qmd

Listado automático (listing):

# En lectures/index.qmd
---
title: "Clases"
listing:
  contents: "semana*"
  type: grid
  sort: "date"
  fields: [date, title, description]
---

Genera automáticamente una cuadrícula o tabla con todos los archivos que coincidan con el patrón. Se actualiza solo cuando agregas nuevos archivos.

Para el blog del curso

El listing tipo default o grid con sort: "date desc" crea automáticamente un blog de actualizaciones del curso.

Temas visuales y personalización

Temas incluidos (Bootswatch):

format:
  html:
    theme: cosmo      # limpio, moderno
    # otras opciones: flatly, litera, lumen,
    # minty, pulse, sandstone, simplex,
    # sketchy, spacelab, united, yeti

Modo oscuro/claro automático:

format:
  html:
    theme:
      light: [flatly, light.scss]
      dark:  [darkly, dark.scss]

El visitante puede alternar entre modos. La preferencia del sistema operativo se detecta automáticamente.

Opciones de formato HTML:

format:
  html:
    toc: true              # tabla de contenidos
    toc-depth: 3
    toc-location: right    # left, right, body
    number-sections: true
    code-fold: true        # código colapsable
    code-tools: true       # botón copiar, ver código
    code-copy: true
    link-external-icon: true
    link-external-newwindow: true
    fig-align: center
    highlight-style: github

code-fold: true en docencia

Permite mostrar el output (figura, tabla) y esconder el código por defecto. Los estudiantes que quieren ver el código hacen clic para expandir.

Renderizar y previsualizar el sitio

Preview en tiempo real:

quarto preview
# → abre el navegador en localhost:PORT
# → se actualiza al guardar cualquier .qmd

Render completo:

quarto render
# → genera _site/ con todos los HTML

# Render solo un archivo
quarto render lectures/semana03/notebook.qmd

Inspeccionar el sitio generado:

ls _site/
# index.html
# lectures/
# labs/
# ...

Opciones de _quarto.yml para eficiencia:

execute:
  freeze: auto      # no re-ejecutar si no cambió
  echo: true
  warning: false
  message: false

# Excluir archivos del render
project:
  render:
    - "*.qmd"
    - "!drafts/*"  # excluir carpeta drafts

En equipos con ayudantes

Cada ayudante trabaja en su rama, hace PR. GitHub Actions renderiza y verifica automáticamente antes del merge.

Open Graph y metadatos para compartir

Cuando compartes el link del curso en redes sociales o WhatsApp, Quarto puede mostrar una vista previa enriquecida.

website:
  open-graph:
    title: "MAT281 — 2025-1"
    description: "Material completo del curso de Aplicaciones de la Matemática en Ingeniería"
    image: assets/img/og-preview.png

  twitter-card:
    title: "MAT281 — 2025-1"
    description: "Material abierto bajo CC BY 4.0"
    image: assets/img/og-preview.png
    card-style: summary_large_image

Metadatos por página:

# Al inicio de cualquier .qmd
---
title: "Semana 3: Regresión Lineal"
description: "Teoría, implementación en Python y aplicación a datos reales"
image: figures/scatter.png
---

Para difusión institucional

Un sitio con metadatos Open Graph correctos se ve profesional cuando se comparte en Twitter, LinkedIn o en el portal de la universidad.
