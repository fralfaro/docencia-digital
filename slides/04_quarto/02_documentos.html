¿Qué es un documento reproducible?

Un documento reproducible es aquel que cualquier persona puede tomar, ejecutar, y obtener exactamente el mismo resultado.

No “similar” — exactamente el mismo.

Para un docente esto significa:

  • El apunte que dice “la media es 5.23” tiene la media calculada desde el código
  • El histograma que muestra esa distribución sale del mismo código
  • Si los datos cambian, ambos se actualizan solos al renderizar

Sin pasos ocultos

❌ “Copié el número del Excel”

❌ “Ajusté la figura manualmente en PowerPoint”

✅ Cada número, figura y tabla se genera desde el código del documento.

Chunks de código: la unidad básica

Sintaxis de un chunk en Quarto:

```{python}
#| label: fig-notas
#| fig-cap: "Distribución de notas del curso"
#| fig-width: 8
#| fig-height: 4
#| echo: false

import matplotlib.pyplot as plt
import pandas as pd

df = pd.read_csv('datos.csv')
plt.hist(df['nota'], bins=20, color='steelblue')
plt.xlabel('Nota')
plt.show()
```

Opciones de chunk más usadas:

Opción Valores Efecto
echo true/false Muestra/oculta el código
eval true/false Ejecuta o no el código
output true/false Muestra/oculta el output
warning true/false Muestra/oculta warnings
fig-cap texto Pie de figura
fig-width número Ancho en pulgadas
label texto ID para referencias cruzadas

Inline code: resultados directamente en el texto

En lugar de escribir el número manualmente:

La nota promedio del curso fue 5.23.

Que puede quedar desactualizado si los datos cambian…

Usa inline code:

La nota promedio del curso fue
`{python} round(df['nota'].mean(), 2)`.

Al renderizar, Quarto ejecuta la expresión y reemplaza el resultado en el texto. Siempre correcto, siempre actualizado.

Más ejemplos de inline code:

El dataset tiene `{python} len(df)` filas y
`{python} df.shape[1]` columnas.

La nota mínima fue `{python} df['nota'].min()`
y la máxima `{python} df['nota'].max()`.

Se identificaron `{python} df['nota'].isna().sum()`
valores faltantes en la columna de notas.

Para apuntes de clase

Cuando actualizas el dataset del siguiente semestre, todos los números del texto se actualizan automáticamente al hacer quarto render.

Figuras y tablas generadas desde código

Figura referenciable:

```{python}
#| label: fig-scatter
#| fig-cap: "Horas de estudio vs nota final"

plt.scatter(df['horas'], df['nota'])
plt.xlabel('Horas de estudio')
plt.ylabel('Nota')
plt.show()
```

En el texto: Como muestra @fig-scatter, existe una correlación positiva...

→ Quarto genera “Figura 1” automáticamente y la numera.

Tabla desde código:

```{python}
#| label: tbl-resumen
#| tbl-cap: "Estadísticas descriptivas por carrera"

resumen = df.groupby('carrera')['nota'].describe()
resumen.round(2)
```

En el texto: La Tabla @tbl-resumen muestra...

Prefijos de las etiquetas

  • Figuras: fig-nombre
  • Tablas: tbl-nombre
  • Ecuaciones: eq-nombre
  • Secciones: sec-nombre

La referencia @fig-nombre genera “Figura 1” (o “Figure 1” en inglés).

Callouts: bloques pedagógicos visuales

Los cinco tipos:

::: {.callout-note}
## Nota
Información adicional de contexto.
:::

::: {.callout-tip}
## Consejo
Buena práctica o atajo útil.
:::

::: {.callout-warning}
## Advertencia
Algo que puede salir mal.
:::

::: {.callout-important}
## Importante
Concepto clave que no debe perderse.
:::

::: {.callout-caution}
## Precaución
Acción que puede tener consecuencias irreversibles.
:::

Opciones de callout:

::: {.callout-tip icon="false" collapse="true"}
## Pista (expandir para ver)

Esta pista solo aparece cuando el estudiante
hace clic para expandir el bloque.
:::

En apuntes de curso

Los callouts son perfectos para: definiciones, ejemplos, advertencias sobre errores comunes, pistas en guías de ejercicios y conexiones con otros temas del curso.

Referencias cruzadas: el sistema completo

Etiquetar:

## Introducción {#sec-intro}

$$
y = \beta_0 + \beta_1 x + \varepsilon
$$ {#eq-regresion}

![Figura de dispersión](scatter.png){#fig-scatter}

Referenciar:

Como vimos en @sec-intro, la ecuación
@eq-regresion modela la relación lineal.

La @fig-scatter muestra el ajuste del modelo.

Quarto genera automáticamente: “Sección 1”, “Ecuación 1”, “Figura 1”.

Configurar el idioma de las referencias:

lang: es
crossref:
  fig-prefix: "Figura"
  tbl-prefix: "Tabla"
  eq-prefix: "Ecuación"
  sec-prefix: "Sección"

Numeración automática

Si insertas una nueva figura antes de la Figura 3, todas se renumeran automáticamente. Las referencias en el texto siguen siendo correctas.

Lo mismo con secciones, tablas y ecuaciones.

Tablas con knitr::kable y gt

Tabla simple con pandas:

```{python}
#| tbl-cap: "Resultados del modelo"

import pandas as pd
from IPython.display import display

resultados = pd.DataFrame({
    'Modelo': ['Lineal', 'Árbol', 'Red'],
    'Precisión': [0.82, 0.91, 0.95],
    'Tiempo (s)': [1.2, 3.7, 45.2]
})

display(resultados.style.format({
    'Precisión': '{:.2%}',
    'Tiempo (s)': '{:.1f}'
}))
```

Tabla desde Markdown puro:

| Semana | Tema | Material |
|--------|------|----------|
| 1 | Introducción | [Ver](lectures/s01.qmd) |
| 2 | Estadística | [Ver](lectures/s02.qmd) |
| 3 | Regresión | [Ver](lectures/s03.qmd) |

Cuándo usar qué

Tabla Markdown: contenido estático (calendario, lista de temas)

Tabla desde código: datos que vienen de un DataFrame o cálculo

Salida multiformato desde un solo archivo

Configurar múltiples formatos:

---
title: "Guía de Laboratorio 3"
format:
  html:
    theme: cosmo
    toc: true
    code-fold: true
  pdf:
    documentclass: article
    geometry:
      - top=2cm
      - left=3cm
  docx:
    reference-doc: plantilla-usm.docx
---

Renderizar todos los formatos:

quarto render guia.qmd
# → guia.html
# → guia.pdf
# → guia.docx

Opciones específicas por formato:

```{python}
#| echo: false
#| fig-format: svg   # SVG para HTML
                     # PDF para PDF (automático)
plt.plot(x, y)
plt.show()
```

El flujo docente ideal

HTML → sitio del curso (navegable)

PDF → imprimible para quien lo prefiera

Word → para colegas que editan en Word
