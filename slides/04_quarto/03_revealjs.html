RevealJS vs PowerPoint vs Beamer

PowerPoint: el archivo .pptx es el resultado. Difícil de versionar, de reutilizar contenido y de incluir código real.

Beamer (LaTeX): produce PDF profesional con matemáticas perfectas. No puede contener código vivo ni animaciones web.

RevealJS (Quarto): una presentación HTML. Se ejecuta en el navegador, puede contener:

  • Código ejecutable con outputs reales
  • Ecuaciones LaTeX
  • Animaciones y transiciones
  • Audio y video embebidos
  • Se publica en GitHub Pages junto con el curso

La ventaja clave para docencia

El mismo archivo .qmd que usas para el apunte puede producir las slides de esa semana.

Un solo lugar de verdad. Cero copiar y pegar.

Highlight de código animado:

```{python}
#| code-line-numbers: "1|2-3|4-6"

import pandas as pd          # línea 1 sola
df = pd.read_csv('datos.csv')# líneas 2-3
df = df.dropna()             # líneas 4-6
df['nota'] = pd.to_numeric(df['nota'])
resultado = df.groupby('carrera').mean()
```

Cada | avanza al siguiente bloque resaltado.

Para explicar código paso a paso

El highlight animado es perfecto para explicar un script línea a línea en clase, con tiempo para que los estudiantes entiendan cada parte.

Código ejecutable en slides

El código corre y el output aparece en el slide:

## Demostración en vivo

```{python}
#| echo: true
#| fig-height: 3

import matplotlib.pyplot as plt
import numpy as np

x = np.linspace(0, 2*np.pi, 100)
plt.plot(x, np.sin(x), label='sin(x)')
plt.plot(x, np.cos(x), label='cos(x)')
plt.legend()
plt.grid(True)
plt.show()
```

Opciones específicas para slides:

execute:
  echo: true       # mostrar código en slides
  warning: false

format:
  revealjs:
    code-block-bg: true    # fondo en bloques de código
    highlight-style: github # tema de resaltado
    fig-align: center

En clase sin internet

Con freeze: auto, los resultados están guardados. La presentación funciona sin necesidad de ejecutar código en vivo — perfecto para conexiones inestables en el aula.

Temas y personalización visual

Temas predefinidos de RevealJS:

Tema Aspecto
default Blanco, clásico
moon Azul oscuro
night Negro
sky Azul claro
serif Elegante con serifa
blood Rojo dramático

Tema personalizado (SCSS):

format:
  revealjs:
    theme: [default, custom.scss]

custom.scss básico:

/*-- scss:defaults --*/
$link-color: #2a7ae2;
$presentation-heading-font: 'Inter', sans-serif;
$presentation-heading-color: #1a1a2e;
$body-bg: #ffffff;

/*-- scss:rules --*/
.title-block {
  font-size: 1.8em;
  font-weight: 700;
  color: #ffffff;
}

.callout-tip {
  border-left: 4px solid #2a7ae2;
}

Colores institucionales

Ajustar $link-color y $presentation-heading-color a los colores UTFSM hace las presentaciones reconociblemente institucionales.

Notas del presentador

Agregar notas al slide:

## Regresión lineal: los supuestos

- Linealidad
- Independencia de errores
- Homocedasticidad
- Normalidad de residuos

::: {.notes}
Explicar que en la práctica, linealidad y
homocedasticidad son los más importantes
de verificar. Los gráficos de diagnóstico
(residuos vs fitted) permiten detectar
violaciones de ambos supuestos.

Mencionar el caso del dataset anscombe
como ejemplo de cómo R² puede ser idéntico
con relaciones completamente distintas.
:::

Activar el modo presentador:

Presionar S durante la presentación → abre ventana de presentador con:

  • Slide actual (lo que ve la audiencia)
  • Siguiente slide (anticipa lo que viene)
  • Notas del presentador
  • Temporizador

Flujo recomendado

  1. Escribe las notas detalladas mientras preparas la charla
  2. En clase, presentas desde el modo presentador
  3. Las notas quedan como guión sin necesidad de otro documento

Publicar la presentación

Render local:

quarto render presentacion.qmd --to revealjs
# → presentacion.html

El .html es autónomo — contiene todo (JS, CSS, fonts) embebido.

Publicar en GitHub Pages:

quarto publish gh-pages

→ Disponible en usuario.github.io/nombre-repo

Los estudiantes acceden desde el navegador en cualquier dispositivo — incluyendo el móvil durante la clase.

Exportar a PDF:

En el navegador con la presentación abierta:

→ Agregar ?print-pdf a la URL

→ Ctrl+P → Guardar como PDF

O desde la terminal (requiere Chromium):

quarto render presentacion.qmd --to revealjs
# → automáticamente también genera PDF
# si tienes chromium instalado

Compartir con estudiantes

URL de GitHub Pages → acceso inmediato, sin descargar nada, funciona en móvil.

Para estudiantes que prefieren PDF → ?print-pdf en el navegador.
