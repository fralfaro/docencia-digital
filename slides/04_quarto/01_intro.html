La promesa de Quarto

Un archivo .qmd, múltiples formatos de salida:

Escribes una vez y renderizas como:

  • HTML → sitio web del curso
  • PDF → para imprimir o enviar a revista
  • RevealJS → presentación de clase
  • Word → informe institucional

Sin copiar y pegar. Sin inconsistencias entre versiones.

Este libro que lees

Este material está construido completamente en Quarto. El sitio web, las presentaciones y los documentos PDF salen del mismo repositorio de archivos .qmd.

Quarto vs Jupyter Notebooks vs R Markdown

Los tres nacen de la misma necesidad: mezclar narrativa con código ejecutable.

Eso se llama programación literaria (literate programming) — propuesta por Donald Knuth en 1984.

Sistema Lenguajes Formatos
Jupyter Python, R, Julia HTML, PDF
R Markdown R, Python HTML, PDF, Word
Quarto Python, R, Julia, Observable HTML, PDF, Word, RevealJS, Docx, y más

La diferencia clave de Quarto

No depende de R. Funciona igualmente bien con Python. Y produce más formatos desde el mismo archivo.

Instalación y configuración

1. Instalar Quarto:

→ quarto.org/docs/get-started

Disponible para Windows, Mac, Linux.

2. Verificar la instalación:

quarto --version
# 1.5.57

3. Instalar la extensión en VS Code:

Extensiones → buscar Quarto → instalar

La extensión agrega: - Preview en tiempo real del documento - Completado de opciones YAML - Resaltado de sintaxis en chunks

Extensiones de VS Code recomendadas:

Extensión Para qué
Quarto Editor y preview
Python Soporte de Python
Jupyter Soporte de notebooks
GitLens Integración con Git

En Overleaf o Colab

Quarto también funciona en la nube. En GitHub Actions lo instalas automáticamente con la action quarto-dev/quarto-actions/setup@v2.

El archivo .qmd: anatomía básica

Frontmatter YAML:

---
title: "Regresión Lineal"
subtitle: "Introducción a modelos predictivos"
author: "Juan Pérez"
date: today
format: html
---

Texto en Markdown:

## Introducción

La regresión lineal modela la relación
entre una variable dependiente $y$ y
variables independientes $x_1, \ldots, x_p$.

$$y = \beta_0 + \beta_1 x_1 + \varepsilon$$

Chunk de código:

```{python}
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv('datos.csv')
df.head()
```

Renderizar:

quarto render documento.qmd
# → documento.html

.qmd = Markdown + YAML + chunks

El formato es familiar si conoces R Markdown o Jupyter. La sintaxis de los chunks es ligeramente diferente pero el concepto es idéntico.

Formatos de salida: uno para cada propósito

HTML (website):

format:
  html:
    theme: cosmo
    toc: true
    code-fold: true

PDF (vía LaTeX):

format:
  pdf:
    documentclass: article
    geometry:
      - top=2cm
      - left=3cm

Word:

format:
  docx:
    reference-doc: plantilla.docx

RevealJS (presentación):

format:
  revealjs:
    theme: default
    slide-number: true
    transition: fade

Múltiples formatos del mismo archivo:

format:
  html: default
  pdf: default
  revealjs: default
quarto render --to html
quarto render --to pdf
quarto render --to revealjs

Un solo archivo fuente

Escribes el contenido una vez. Cambiar de HTML a PDF es solo cambiar el format en el YAML.

El ciclo de trabajo con Quarto

quarto preview es tu mejor amigo

Abre el documento en el navegador y lo actualiza automáticamente cada vez que guardas. Es como el modo de edición en Google Docs pero para publicación académica.

Opciones de ejecución de código

A nivel de documento (en el frontmatter):

execute:
  echo: true       # mostrar el código
  warning: false   # ocultar warnings
  message: false   # ocultar mensajes
  freeze: auto     # no re-ejecutar si no cambió

A nivel de chunk individual:

```{python}
#| echo: false      # ocultar este código específico
#| fig-cap: "Distribución de notas"
#| fig-width: 8
#| fig-height: 4

plt.hist(df['nota'])
plt.show()
```

Opciones más usadas en docencia:

Opción Efecto
echo: false Muestra solo el output
echo: true Muestra código + output
eval: false Muestra el código sin ejecutar
output: false Ejecuta pero no muestra output
fig-cap: "..." Pie de figura
code-fold: true Código colapsable

freeze: auto es crítico para deployment

Con freeze, Quarto guarda los resultados. En GitHub Actions, solo re-ejecuta el código que cambió. El tiempo de build puede reducirse de 15 min a 2 min.

Tipos de proyecto Quarto

Website (sitio del curso):

quarto create project website mi-curso
mi-curso/
├── _quarto.yml    # configuración del sitio
├── index.qmd      # página de inicio
├── lectures/
│   └── semana01.qmd
└── labs/
    └── lab01.qmd

Book (libro o apuntes):

quarto create project book mi-libro

Genera automáticamente capítulos numerados con navegación.

Blog:

quarto create project blog mi-blog

Posts con fecha, categorías y RSS automático.

Manuscript (paper):

quarto create project manuscript mi-paper

Optimizado para publicación académica con múltiples formatos de salida.

¿Qué proyecto elegir para docencia?

Website → sitio del curso completo con navegación

Book → apuntes o libro de texto del curso

RevealJS (sin proyecto) → presentaciones de clase individuales

La ventaja sobre PowerPoint: reproducibilidad

Con PowerPoint:

  • La figura de regresión la hiciste manualmente en Excel
  • Si los datos cambian, actualizas Excel, exportas PNG, lo insertas de nuevo
  • El número “media = 5.23” del texto puede estar desactualizado
  • Envías el .pptx a un colega → no puede reproducir los cálculos

Con Quarto:

  • La figura sale del código que está en el mismo documento
  • Si los datos cambian, quarto render → todo actualizado automáticamente
  • El número “media = {python} df['nota'].mean():.2f” siempre es correcto
  • El colega clona el repositorio y reproduce todo en segundos

El principio de reproducibilidad

El documento es la verdad. No hay pasos ocultos entre los datos y el resultado final.

Eso tiene valor en docencia: los estudiantes ven exactamente cómo se llegó a cada número y cada figura.
