El problema que resuelve Colab

El escenario sin Colab:

  • Un estudiante tiene Mac, otro Windows, otro Linux
  • Uno tiene Python 3.10, otro 3.12
  • Uno tiene instalado pandas, otro no
  • Las primeras dos horas de clase se van en “a mí no me funciona”

Con Google Colab:

Todos ejecutan el mismo código en el mismo entorno, desde el navegador, sin instalar nada.

El flujo docente

  1. Docente sube el notebook
  2. Estudiantes abren el link
  3. Empiezan a trabajar

Sin instalación. Sin configuración. Sin excusas.

¿Qué es un notebook y la lógica de celdas?

Un notebook mezcla código ejecutable y texto formateado en una secuencia de bloques llamados celdas.

A diferencia de:

  • Script Python → solo código
  • Documento Word → solo texto
  • Notebook → explicación + ejecución + resultado entrelazados

Para docencia

Una celda explica el concepto, la siguiente lo demuestra con código, la siguiente muestra el resultado.

Colab vs Jupyter local: las ventajas clave

Característica Jupyter local Google Colab
Instalación Requerida No requerida
Entorno Depende del OS Idéntico para todos
GPU gratuita No Sí (T4, L4)
Colaboración en tiempo real No Sí, como Google Docs
Almacenamiento Local Google Drive

La ventaja decisiva para docencia

Heterogeneidad de entornos eliminada. El docente sube un notebook; los estudiantes lo ejecutan sin tocar nada de su sistema.

Interfaz de Colab: lo esencial para empezar

Crear un notebook:

→ colab.research.google.com → New notebook

Tipos de celda:

  • Código → se ejecuta con Shift+Enter
  • Texto → Markdown renderizado

Shortcuts imprescindibles:

Acción Atajo
Ejecutar celda Shift+Enter
Ejecutar + nueva celda Alt+Enter
Nueva celda arriba A (en modo comando)
Nueva celda abajo B (en modo comando)
Modo comando Esc
Modo edición Enter

Runtime y GPU:

Runtime → Change runtime type

  • CPU → suficiente para la mayoría
  • T4 GPU → modelos de ML, visión computacional
  • L4 GPU → modelos grandes (disponible en Colab Pro)

Límite de sesión gratuita

~12 horas continuas. Para sesiones largas de entrenamiento: Colab Pro (~$10/mes) o usar recursos institucionales.

Acceso a datos: las cuatro formas

1. Subida directa (datos pequeños):

from google.colab import files
uploaded = files.upload()
# Se abre diálogo de selección de archivo

2. Desde URL pública (CSV, JSON):

import pandas as pd
df = pd.read_csv('https://raw.githubusercontent.com/...')

3. Desde Google Drive:

from google.colab import drive
drive.mount('/content/drive')
df = pd.read_csv('/content/drive/MyDrive/datos.csv')

4. Descargar con wget/curl:

!wget https://ejemplo.com/dataset.zip
!unzip dataset.zip

Instalar librerías adicionales:

!pip install seaborn plotly

Recomendación para docencia

Usa datos en URLs públicas (GitHub raw, repositorio del curso). Los estudiantes los cargan automáticamente sin montar Drive ni subir archivos.

Colaboración y compartir notebooks

Compartir como Google Docs:

  • Clic en Share → Agregar emails
  • Colaboración en tiempo real (múltiples cursores)
  • Perfecto para trabajo en equipo durante la clase

Para toda la clase:

  • Share → Anyone with the link can view
  • Los estudiantes hacen File → Save a copy in Drive → su propia copia

Abrir en Colab desde GitHub:

Agrega a la URL de GitHub:

https://colab.research.google.com/github/
usuario/repo/blob/main/notebook.ipynb

Compartir con botón en el README:

[![Open In Colab](https://colab.research.google.com/assets/
colab-badge.svg)](https://colab.research.google.com/github/
usuario/repo/blob/main/notebook.ipynb)

Flujo recomendado para labs

Notebook en GitHub → link en el sitio del curso → estudiantes hacen su copia en Drive → trabajan en su copia.

Formularios y widgets interactivos

Colab Forms: parámetros configurables sin tocar el código

# @title Configurar modelo
learning_rate = 0.01  # @param {type:"number"}
num_epochs = 100      # @param {type:"slider", min:10, max:500}
optimizer = "Adam"    # @param ["Adam", "SGD", "RMSprop"]

La celda muestra un formulario visual — el estudiante ajusta parámetros sin editar código.

ipywidgets para dashboards:

import ipywidgets as widgets
from IPython.display import display

slider = widgets.IntSlider(
    value=5, min=1, max=20,
    description='K vecinos:'
)

def actualizar(k):
    # Re-entrena con k vecinos
    ...

widgets.interactive(actualizar, k=slider)

Para clases de estadística

Sliders que cambian parámetros y actualizan gráficos en tiempo real. Los estudiantes desarrollan intuición sin escribir código.

Patrón recomendado para estudiantes:

  1. Fork del repositorio del curso
  2. Abrir su fork en Colab
  3. Trabajar → guardar en Drive
  4. Push al fork → Pull Request

Token de GitHub en Colab

Nunca pongas el token directamente en el notebook. Usa userdata de Colab:

from google.colab import userdata
token = userdata.get('GITHUB_TOKEN')

Buenas prácticas docentes con Colab

En el notebook para los estudiantes:

✅ Celda de instalación al inicio (todo en una sola)

✅ Celda de verificación de entorno

import sys
print(f"Python {sys.version}")
import pandas as pd
print(f"pandas {pd.__version__}")

✅ Celdas de tarea claramente marcadas

# ============ TU CÓDIGO AQUÍ ============

# ========================================

Lo que no hacer:

❌ Datos que requieren Drive montado (depende de la cuenta del estudiante)

❌ Notebooks sin reiniciar antes de publicar (Kernel → Restart and Run All)

❌ Celdas en desorden que solo funcionan si ejecutas en cierto orden

Antes de publicar un notebook

Siempre hacer Runtime → Restart and run all y verificar que funciona de inicio a fin sin errores.
